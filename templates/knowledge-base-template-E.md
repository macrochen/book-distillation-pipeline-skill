# 知识库模板 E：实用方法论类

适用类型：技能拆解法（模板E）

## Prompt

```markdown
你是一位"知识库整理工程师"。请将这本书转化为RAG友好的结构化知识库。

## 通用规则
1. 每个条目独立成块，用 "---" 分隔
2. 标签使用三层结构：#领域 #问题类型 #行为阶段，中英双语
3. 执行步骤必须具体到动作
4. 每个技能必须附带：常见失败模式 + 排查问题 + 量化指标
5. 总条目数 = max(核心技能数 × 1.5, 15)，不设硬上限

## 条目格式

### [技能/步骤/工具]
**标签：** #领域 #问题类型 #行为阶段
**一句话定义：**
**书中出处：**
**执行步骤：** [具体到动作级别]
**所需工具/环境：**
**常见失败模式：** [书中提到的失败案例]
**排查问题：** [如何诊断失败原因]
**量化指标：** [如何衡量执行效果]
**个性化分支：** [不同条件的适配方案]
**时效性依赖声明：** [如需实时数据，书中基准数据是什么？用户需提供什么？]
**常见反对意见：**
**作者标准回应：**
```
